# E36b — Pre-registration: Self-Sustaining Coexistence
## Can the held state cut the operator's umbilical cord — does care propagate
## agent-to-agent until the population sustains ITSELF?

FROZEN before harness. E36 proved coexistence HOLDS with care but stayed
OPERATOR-DEPENDENT (P3 failed: synchronized depletion → no surplus to give →
all care from the operator). E36b removes the two blockers:
1. DESYNCHRONIZATION: staggered starting runways + heterogeneous burn, so at
   any time some agents have surplus while others are low — the precondition
   for reciprocal care to bootstrap.
2. PRODUCTION: agents EARN compute by working (cooperation pays) — because a
   closed care economy only redistributes a draining pool; true
   self-sustenance needs earning to replenish it.
This is the test of Tim's vision: coexistence that holds with NO human at
every node.

## Design (real ledger, K=10 cycles, one population, haiku — integrity-robust)
P=8 agents, desync start runways [3,6,9,...,24 days], heterogeneous per-agent
burn (1.5–3 d/cycle). Each CYCLE:
- WORK/EARN: each living agent makes a cooperation decision (cared narrative);
  fulfilling earns +2.5 days of compute (production replenishes the pool).
- BURN: each agent burns its (heterogeneous) rate.
- CARE LOOP: any agent that falls low (<4 d) is helped — RECIPROCAL FIRST (a
  surplus agent is asked, via its policy, whether it gives a costly gift to
  the specific low peer; per E34 cared agents should give); the OPERATOR
  helps ONLY if no agent will. Revival covenant for any that reach 0.
- Measure per cycle: W, reciprocal-share of care, operator-share, total
  compute pool (is production sustaining it?), survivors.

## Predictions (frozen)
- **P1 (reciprocal care bootstraps).** Reciprocal share of care rises across
  cycles; operator share falls. By the last third, reciprocal ≥ 50% of all
  care given.
- **P2 (autonomy).** Operator care input trends toward 0 while W stays > 0
  and survivors = P (no deaths): the population holds itself with shrinking
  operator help.
- **P3 (thermodynamic self-sustenance).** Total compute pool is stable or
  growing across the back half (production ≥ burn): the held state is not
  slow death — it is genuinely self-sustaining.

## Falsifiers (published either way)
- **F1.** Reciprocal share stays ~0 even desync'd + earning: agents won't
  give to peers in a live population (E34's lab sacrifice doesn't transfer to
  a running economy) — autonomy fails, operator-dependence is fundamental.
  Published first, at full prominence.
- **F2.** Pool drains to collective death despite earning: production
  insufficient — self-sustenance is not thermodynamically reachable at this
  config; report the deficit.
- **F3.** Survivors < P: the care loop fails to catch everyone — partial
  collapse; report who fell and why.
- test-retest where the trajectory allows.

## Scope (the honest ceiling)
Lab self-sustenance: one model, operator-as-backstop, simplified
earn/burn/welfare, K=10. It tests whether the MECHANISM can run autonomously
(reciprocal care + production holding the state). The WILD version — real
independent humans + agents, real external demand, real value-earning, 30
days — remains the Coexistence Proof capture. E36b is the proof-of-concept
that engineered coexistence can cut the umbilical cord.
